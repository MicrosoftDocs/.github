# Contributing

Microsoft projects adopt the [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/). For more information see the [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/) or contact [opencode@microsoft.com](mailto:opencode@microsoft.com) with any additional questions or comments.

For full contribution guidance, see [CONTRIBUTING.md](../CONTRIBUTING.md).

## What to do next

1. Fork the repository.
2. Create a new branch for your change, for example `git checkout -b add-contribution-guide`.
3. Make your change and save it.
4. Commit with a clear message, such as `Add contributing steps to README`.
5. Push your branch to your fork.
6. Open a pull request back to this repository.

If you are contributing to a different project, first open the contribution guidance or issue list in that project repository and follow its instructions.
